## 对里程碑的影响

成功的对话与错误探测确认了网关现有对话实现所依赖的上游信封。工具结果观察不会
扩大网关公开支持的消息角色子集；该限制仍是已记录的兼容差异。

明确发起的后续运行已经确认图片 URL 输出契约。当天核对的 Agnes 文档说明文生图
Base64 应使用 `return_base64: true`，但两次成功请求均返回 URL，且
`data[0].b64_json: null`。使用 `extra_body.response_format: "b64_json"` 的定向
请求返回了文档描述的 Base64 信封。因此，网关提交 `daecebb` 将 OpenAI 标准字段
映射到这个已验证控制，不增加自动回退，也不会创建第二个可能计费的请求。

独立执行的 Data URI 图片编辑探测已经通过，确认 Agnes 接受 `extra_body.image` 与
相同的 Base64 控制。经历两次临时 `503` 后，正式 `image-base64` scope 已经基于
提交后的网关源码通过，结构为 `b64_json: string` 与 `url: null`。

M2 的全部实时 scope 现已通过：Chat JSON/SSE/工具、安全错误信封、图片 URL/Base64
输出和 Data URI 图片编辑。结合普通网关与 SDK 测试，M2 正式完成。探测器会在断言
