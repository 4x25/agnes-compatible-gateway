## Milestone impact

The first live observation showed that creation succeeded but the documented
legacy `/v1/videos/{task_id}` lookup rejected its new task. Agnes creation also
returned a separate `video_id`. The accepted run confirmed that this identity is
directly retrievable through the recommended stateless `/agnesapi` endpoint.

Commit `21b8e70` therefore exposes Agnes `video_id` as the OpenAI-style public
`id`, preserves `task_id` as an extension, and uses the recommended video-ID
query for retrieval and content resolution. A single read-only legacy fallback
on `400`/`404` keeps IDs returned by older gateway versions usable. This remains
fully stateless and adds no database, cache, queue, or generation retry.

The committed implementation passed formatting, lint, type checking, all 42
gateway/SDK/OpenAPI/Fresh tests, production builds, and the complete Chromium
smoke suite before this live run. Real creation, terminal polling, completed
media URL resolution, and byte-range download have now all passed, completing
M3.
