## 浏览器与 API 覆盖

外部 Chromium 运行有意不在测试台提交生成请求，因为远端托管网关无法安全改指向本地
假 Agnes；真实部署上的 UI、无障碍、响应式和凭据内存检查全部通过。随后，具有独立
安全开关的部署探测使用一次性调用方 Key，通过同一 Origin 直接执行了全部平台敏感的
公开工作流。

图片编辑跨越 Deno Deploy 请求生命周期完成，确认平台可以承载 multipart 上传与较长
上游响应。视频创建、重复无状态查询、终态媒体解析和公开字节范围代理也全部完成。
每个生成请求只执行一次；只有幂等视频读取会重复。

## 对发布的影响

此前的[候选版本工作流](https://github.com/4x25/agnes-compatible-gateway/actions/runs/29655101517)
发布了可匿名拉取、包含 `linux/amd64` 与 `linux/arm64` 的 OCI Index。每种架构均有
独立 SPDX 与 SLSA 证明 Manifest，以无特权 `deno` 用户运行，并将不可变源码 SHA
写入 `DENO_DEPLOYMENT_ID`。工作流检查远端 Index 后，以只读文件系统、移除全部
capabilities 的方式成功启动已发布 digest。

真实 Deno Deploy 与 GHCR 候选版本共同满足 M5 的部署和容器验收门槛。在版本化源码
打 Tag、最终多架构镜像验证完毕且公开 GitHub `v0.1.0` Release 存在前，README 中的
M5 仍保持进行中。
