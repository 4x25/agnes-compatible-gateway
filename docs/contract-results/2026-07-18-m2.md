## Milestone impact

The successful Chat and error probes confirm the upstream envelopes exercised by
the gateway's existing Chat implementation. The tool-result observation does not
expand the gateway's public message-role subset; that remains a documented
compatibility gap.

An explicit follow-up confirms the Image URL-output contract. The Agnes
documentation checked on this date says that text-to-image Base64 uses
`return_base64: true`, but two successful requests returned URLs with
`data[0].b64_json: null`. A targeted request using
`extra_body.response_format: "b64_json"` returned the documented Base64
envelope, so gateway commit `daecebb` maps the standard OpenAI field to that
verified control. It does not add an automatic fallback or a second billable
request.

The separately requested Data-URI image-edit probe passed and confirms Agnes
accepts `extra_body.image` with the same Base64 control. The formal
`image-base64` scope was updated to the verified mapping but both committed
reruns ended with upstream `503` responses. M2 therefore remains in progress
until that scope completes successfully in a later controlled run. The probe
records a bounded response shape before assertions, so contract drift can be
diagnosed without retaining response values.
