## Milestone impact

The successful Chat and error probes confirm the upstream envelopes exercised by
the gateway's existing Chat implementation. The tool-result observation does not
expand the gateway's public message-role subset; that remains a documented
compatibility gap.

An explicit follow-up confirms the Image URL-output contract. The Agnes
documentation checked on this date still says that text-to-image Base64 uses
`return_base64: true`, but one successful request did not contain a non-empty
`data[0].b64_json`; later diagnostic requests ended with upstream `503`, `504`,
and `504` responses. The gateway therefore keeps the documented mapping without
adding an automatic fallback or a second billable request.

The separately requested Data-URI image-edit probe passed and confirms Agnes
accepts `extra_body.image` with `extra_body.response_format: "b64_json"`. M2
must remain in progress only until text-to-image Base64 completes successfully
in a later controlled run. The live probe records a bounded response shape
before asserting either Base64 field, so future contract drift can be diagnosed
without retaining response values.
